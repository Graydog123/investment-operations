# 第九章　仓位不是态度，是经营规模

你买了多少，不是在向市场表达多有信心，而是在决定判断错误时要付多少钱。

## 冰箱为什么不能第一天就塞满

一家小餐馆准备推出一道新菜。老板亲自试过味道，厨师也很有把握，几个熟客试吃后都说不错。老板越想越兴奋：既然菜好、评价也好，不如一次把一个月的食材全部买齐。批量采购还能便宜一些，真卖火了，也不怕临时缺货。

可新菜端上菜单以后，顾客只是偶尔点一份。有人嫌辣，有人觉得价格高，还有人根本没注意到它。几天后，冰箱被新食材占满，原来的畅销菜反而没地方备货。老板开始打折、送小菜、催服务员重点推荐。再后来，他研究的已经不是顾客喜不喜欢，而是怎样证明自己当初没有进错货。

问题出在菜不好吗？未必。问题出在老板太有信心吗？也不完全是。真正的问题是：他把对一道新菜的判断，直接翻译成了一个月的采购量，中间没有经过“卖不掉时愿意报废多少”的计算。

股票账户也常犯同样的错。研究得越深入，人越容易觉得应该重仓；越看好一家公司，越愿意把仓位当成信念证明。可观点只回答“为什么可能上涨”，仓位回答的是另一件事：如果判断没有按计划兑现，账户要承受多大损失，还剩不剩重新选择的能力。

> 仓位不是表达信心，而是表达愿意承担多少错误。

## 仓位不是观点的音量

很多人把仓位理解成观点的音量。三成仓说明一般看好，五成仓说明非常看好，满仓说明确信无疑。这个想法听起来自然，却把两件不同的事混在了一起。

信心来自主观判断，仓位属于账户经营。判断再有把握，也可能遇到未被发现的信息、突发事件、市场风格变化或流动性收缩。仓位的任务不是证明判断有多坚定，而是把这些未知翻译成一个账户仍能承受的尺寸。

同样看好一家公司，两个人的合理仓位可能完全不同。一个人使用长期闲钱，组合里没有相近资产，标的成交活跃，失效条件距当前价格不远；另一个人正在融资，其他持仓也依赖同一产业景气，标的波动很大，真正失效时又可能难以成交。两个人的观点可以一样，经营规模却不能一样。

因此，仓位不能从“我有多看好”直接推出。它至少要经过五个刻度：账户愿意损失多少、失效点离现在多远、其他持仓会不会一起出事、退出时会不会打折，以及杠杆会不会放大整个过程。

这五个刻度共同形成错误预算。它不是预测最大亏损，也不是承诺亏到某个百分比一定卖出。它是事前给错误划出的账户边界：这笔判断即使失败，最多可以占用多少净值、注意力和未来选择。

## 先算报废额，再决定进多少货

餐馆采购新食材，先问的通常不是“这道菜最多能卖多少”，而是“如果卖不动，最多能报废多少”。投资也应当采用同样的顺序：先写账户允许付出的错误成本，再倒推可以买多少。

最简化的关系是：

> 可用仓位，约等于账户错误预算除以单笔失效幅度，再根据相关性、流动性和杠杆向下调整。

假设账户有一百个资本单位，愿意为一次普通判断错误支付一个单位；从买入位置到真正失效，预计会损失持仓价值的十分之一。那么，在暂不考虑其他风险时，这笔仓位的理论上限是十个资本单位。若组合里还有几只会同时受损的同类股票，或者真正退出时可能多损失一截，实际可用仓位就应低于十个单位。

这个例子不是建议把单笔损失固定为1%，也不是告诉读者止损必须设在10%。它只展示一种顺序：损失预算在前，仓位在后。不同策略的失效条件、波动路径和持有期限不同，数字自然不同。

| 刻度 | 先问什么 | 对仓位的影响 |
| --- | --- | --- |
| 错误预算 | 这笔判断失败，账户最多愿意损失多少？ | 预算越小，可用规模越小 |
| 失效幅度 | 从当前价格走到真正失效，可能损失多少？ | 失效距离越远，可用规模越小 |
| 相关性 | 其他持仓会不会因同一原因一起受损？ | 同因子越集中，单股预算越要收缩 |
| 流动性 | 想退出时，能否按计划成交？ | 滑点和成交困难越大，可用规模越小 |
| 杠杆 | 下跌是否会提高实际杠杆或触发被迫行动？ | 杠杆越高，容许规模越小 |

专业上，这组检查对应风险预算、头寸规模、相关暴露、市场冲击成本和杠杆敏感度。生活里的“报废额”，并没有取代这些金融概念，只是让计算顺序变得更直观。

## 失效不是“跌得让我难受”

要用错误预算反推仓位，必须先说明什么叫失效。这里最常见的偷懒，是把成本价附近的某个跌幅直接当成止损线。可真正的失效不来自情绪，而来自买入理由。

如果你买的是平台突破，失效可能是股价重新跌回平台并无法收复；如果你买的是主线核心，失效可能是板块地位下降、个股持续弱于同类核心；如果你买的是产品验证，失效可能是验证延期、客户导入不及预期，或者新证据停留在原来的等级。不同交易身份，对应不同的失效距离。

失效距离越模糊，仓位越不能大。因为“我会看情况处理”在下跌发生后，常常会变成不断移动的终点。今天说跌破平台退出，明天改成等二十日线，后天又改成行业长期空间仍大。没有清晰失效条件时，真实风险不是某个固定跌幅，而是理由可以无限延长。

这也是为什么本书不提供通用仓位表。对一只波动小、流动性好、失效条件清楚的资产，较大仓位可能仍在预算内；对一只跳空风险高、事件结果二元、成交又不充分的资产，即使仓位看起来不大，失效时也可能超出预算。

> 说不清什么时候算错，就算不清自己买得起多少。

## 同一冰箱里的菜，要合并计算

一个老板不会因为牛肉分装在五个盒子里，就认为自己只压了一盒库存。账户里持有五只股票，也不代表风险被分成了五份。若它们都依赖同一批顾客、同一轮资本开支、同一种商品价格或同一种市场风格，它们可能在压力来临时一起变质。

因此，错误预算不能只按股票逐只计算，还要按共同失效原因计算。两只存储股、两只半导体设备股和一只消费电子股，行业标签看起来不同，资金交易时却可能都属于高估值科技成长。平时相关性不高，不代表退潮时不会挤进同一条出口。

逐股看，每一笔仓位也许都“可以承受”；合并看，同一主线或同一风格的总预算可能已经超支。这就像每道菜只备了三天货，但十道菜都依赖同一种容易腐坏的原料。真正应当限制的不是盒子数量，而是同一种错误能够造成的总损失。

所以，一只新股票进入组合时，不能只问“它好不好”，还要问“账户里是否已经有了同一种生意”。如果新仓只是重复备货，它即使单独看很优秀，也未必值得占用新的错误预算。

## 观察菜与招牌仓

一家餐馆推出新菜，通常不会第一天就把它印在菜单最显眼的位置。老板会先少量供应，观察点单、复购、退菜和毛利，再决定是否长期备货。本书把这种用有限成本购买真实反馈的仓位叫作观察菜。

观察菜不是胆小仓，也不是随便买一点求心理安慰。它必须承担明确任务：让未知暴露出来。买入前写下要观察什么，买入后检查市场、公司和自身情绪给出了什么新信息。如果没有观察问题，只是因为怕踏空而先买一点，这笔仓位买到的不是信息，而是焦虑的暂时缓解。

招牌仓则不是“最喜欢的股票”。它是已经经过多轮证据检验、能够承载较大资本、并在组合里承担主要收益责任的持仓。招牌仓仍然需要错误预算和退出条件，只是它获得了更成熟的证据、更好的赔率或更高的市场确认。

观察菜与招牌仓都不是固定比例。对不同账户、不同策略和不同流动性，它们可能对应完全不同的数字。真正稳定的不是百分比，而是升级规则：未知较多时用小规模购买信息；证据成熟后才扩大经营；证据退化时，角色也要降级。

> 小仓不是因为不敢判断，而是因为真实反馈还没有卖给你。

## 加仓前，要通过三道阀门

价格下跌以后，人最容易产生一种错觉：同样的公司变便宜了，所以理应买得更多。问题是，下跌只改变了价格，并没有自动改善证据。顾客不点的菜就算食材降价，也不等于应该继续塞满冰箱。

加仓至少要通过三道阀门。

第一道是证据阀：新增了什么可以独立核验的事实？是产品从送样进入验证，还是从验证进入量产？是主线在分歧后重新获得资金确认，还是只有一篇旧消息被再次传播？没有新增证据，就不能把重复叙事写成升级。

第二道是赔率阀：价格变化以后，潜在回报与失效损失是否真的改善？价格下降可能提高赔率，也可能意味着市场正在计入此前没有重视的坏消息。证据没有恶化、隐含预期明显下降，才有讨论空间；不能因为浮亏让人不舒服，就把“更想回本”误写成“赔率更好”。

第三道是预算阀：即使证据或赔率改善，同一因子的剩余错误预算还够不够？加仓后若一旦失效，组合损失是否仍在边界内？是否会挤掉现金、提高融资压力或让退出变得困难？好机会也不能使用已经花完的预算。

三道阀门不是要求每次都等到所有信息完全确定。完全确定时，价格通常也不会便宜。它们只是要求加仓必须有一个与“我已经亏了”不同的新理由。

| 检查门 | 允许通过的理由 | 不能通过的理由 |
| --- | --- | --- |
| 证据阀 | 可核验事实升级，或市场地位重新确认 | 传闻、旧信息复述、自己研究得更久 |
| 赔率阀 | 预期下降而证据未恶化，或潜在回报上升 | 只因股价低于成本、想尽快回本 |
| 预算阀 | 同因子仍有损失额度，退出能力未恶化 | 已经超配，却用“越跌越便宜”继续加 |

> 加仓应该来自证据升级，不应来自价格让人不舒服。

## 仓位越大，越会改变你看到的事实

仓位不只放大盈亏，还会改变观察者。持仓很小时，人容易冷静列出反证；仓位很大时，同一条坏消息会被迅速解释成市场误读，同一条普通消息则可能被看成重大催化。研究从寻找事实，慢慢变成维护仓位。

这不是简单的意志薄弱。大额盈亏会占用注意力，缩短时间感，放大损失厌恶。人会更频繁地看盘，更急于寻找认同，也更难承认原假设已经变化。于是，仓位超过心理承载能力后，连最初用来判断它的那套研究能力也会下降。

当你发现自己无法阅读反方材料、不断刷新价格、睡眠被持仓打断，或者所有研究都在寻找继续持有的理由时，问题可能已经不是公司，而是经营规模。此时减仓不一定代表看空，它可能只是把仓位降回能够正常思考的尺寸。

专业机构可以用团队分工、风险系统和强制流程减轻这种扭曲，但不能彻底消除。个人投资者更需要承认：心理资本也是错误预算的一部分。一个理论上能承受的仓位，如果已经让人无法执行规则，实际就已经过大。

## 三种规模，三种结果

### 伯克希尔：流动性让规模仍有回旋余地

伯克希尔·哈撒韦在2023年年报中表示，公司持有远超通常认知所需的现金和美国国库券；回顾2008年时，公司依靠经营产生现金，没有依赖商业票据、银行授信或债务市场。[1] 同一份年报还强调充足流动性的重要性，并披露了大额现金、现金等价物和美国国库券。

这个案例不提供个人账户应持有多少现金的标准。伯克希尔拥有保险浮存金、多元经营业务和与散户完全不同的负债结构，不能照搬。它说明的只是一个正向机制：规模很大不必然失控，前提是流动性、负债和未承诺资本使经营者仍有调整空间。

真正值得学习的不是某个现金比例，而是次序。伯克希尔不是等危机发生以后才临时寻找退出通道，而是在平常就把生存和选择写进规模。看起来“闲着”的流动性，使外部市场失灵时仍然保留主动权。

### Zillow：小误差进入大库存

Zillow Offers把房屋估值、买入、翻修和出售连接成规模化业务。2021年11月，Zillow宣布逐步关闭该业务。公司表示，房价预测的不确定性远超预期，继续扩张会带来过大的盈利和资产负债表波动；当季还确认了约3.04亿美元库存减记，并提到翻修与转售能力约束。[2]

这不是一句“算法错了”就能概括的故事。预测误差只有进入真实库存，才会与采购价格、翻修速度、融资、周转和市场承接一起发生作用。规模越大，一个小偏差越可能在多个环节同时放大。

房屋库存与股票仓位当然不相同。房屋需要翻修和逐套出售，股票通常具有更连续的报价。这个案例只用于说明：判断的精度不能脱离经营规模。一个在小规模试验中可接受的误差，放大以后可能超过资产负债表和运营能力。

### Archegos：看似很多头寸，实际共用一条出口

美国证券交易委员会在2022年的起诉公告中称，Archegos通过总收益互换和保证金交易建立了高度集中的巨大股票敞口；按SEC的指控，其价值和总敞口在约一年内从约15亿美元和100亿美元，扩大到峰值超过360亿美元和1600亿美元。集中持仓下跌后触发保证金追缴，Archegos无法满足，随后违约并给交易对手造成数十亿美元损失。[3]

这个案例涉及监管指控中的欺诈、误导交易对手和市场操纵，不能被简化成“集中持仓必然失败”。但它极端地展示了规模、集中、杠杆和流动性如何相互强化：账面上可以有多个合约和多个交易对手，底层却依赖少数股票和同一批买盘；一旦价格反向，所有退出要求会同时出现。

Archegos的问题不只是看错了几只股票，而是经营规模让错误没有缓冲。上涨时，规模制造更多购买能力；下跌时，规模制造保证金需求和集中卖出。同一种结构在两个方向上都自我强化，直到账户失去选择。

## 减仓不是平均切掉每一盘菜

当组合的错误预算已经超支，最没有用的做法是对所有持仓机械减去相同比例。经营者真正要处理的是：哪些仓位最缺证据，哪些仓位重复占用同一种预算，哪些仓位最难退出，以及哪些负债正在缩短时间。

通常的顺序是先停止继续扩张，再处理融资和短期义务；随后减去逻辑失效、相对强度下降或证据长期停滞的持仓；再处理重复暴露和流动性较差的部分。核心仓可以比观察仓获得更多耐心，但没有永久豁免。如果主线、公司或账户层面的失效已经发生，所谓核心只是一段历史身份。

“先减弱者，再减核心”是一种常见起点，不是机械法律。若核心仓本身占用了绝大部分预算，或者风险事件只发生在核心上，就必须直接处理核心。减仓的目的也不是让持仓数量看起来平均，而是把总损失重新压回账户可以承受的范围。

退出之后应保留重新进入机制。卖出不是宣布这家公司永远不能买，而是取消旧仓位对未来资本的占用。当证据重新升级、价格重新提供赔率、错误预算重新可用时，可以重新评估。这样，减仓就不再是一场尊严审判，而是一项规模调整。

## 买入以前，先填“仓位卡”

下一次准备下单，不要先写准备买几成。先把下面几栏写完，再让数字出现。

| 项目 | 必须回答的问题 |
| --- | --- |
| 交易身份 | 我买的是趋势、事件、估值修复，还是长期经营结果？ |
| 失效条件 | 哪个可观察事实出现，说明原假设不再成立？ |
| 失效幅度 | 从计划买入位置到可执行退出位置，正常情况下可能损失多少？ |
| 错误预算 | 这笔交易和这个共同因子，最多可以损失多少账户净值？ |
| 相关暴露 | 哪些现有持仓会因同一原因一起受损？ |
| 退出能力 | 普通日和压力日分别需要多久退出，滑点可能多大？ |
| 杠杆影响 | 下跌会不会提高实际杠杆、触发追缴或占用生活现金？ |
| 初始角色 | 这是观察菜、普通持仓，还是已有充分证据的招牌仓？ |
| 升级条件 | 新增什么证据，才允许继续加仓？ |
| 降级条件 | 哪些变化出现时，应减仓、退出或重新等待？ |

如果失效条件写不出来，或者退出能力无法估计，只能使用最小观察规模，甚至暂时不买。如果加仓理由里只有“跌了很多”“成本更低”或“不想卖在底部”，加仓阀没有通过。如果单股预算没有超支，但同一主线的总预算已经花完，也不能再用新的股票代码绕过限制。

仓位卡的价值不在于计算出一个看似精确的答案。它的价值是强迫信心接受账户约束，让每一次规模变化都能被解释、被检查、被复盘。

## 经营规模，决定错误能活多久

投资者无法消灭错误。研究再深入，也会遇到未知；趋势再清晰，也可能突然反转；估值再便宜，也可能继续变便宜。成熟系统与冲动交易的区别，不是谁永远正确，而是谁在错误发生以前就决定了它能造成多大伤害。

小仓不天然高明，重仓也不天然鲁莽。证据成熟、赔率充足、相关性可控、退出顺畅并且账户仍保留经营权时，集中可能是合理经营；反过来，一堆看似分散的小仓若共同依赖同一种风格，也可能合成一个巨大错误。

真正的纪律不是永远买得少，而是让规模与证据同步。观察菜用有限成本购买反馈，招牌仓用成熟证据承担收益责任；证据升级，仓位才有资格升级；证据退化，旧身份不能阻止降级。

下一次想说“我对这只股票很有信心”时，再多问一句：如果我错了，准备付多少钱？只有这个问题有了答案，信心才真正进入经营。

> 先决定错误可以有多大，再决定观点可以买多少。
