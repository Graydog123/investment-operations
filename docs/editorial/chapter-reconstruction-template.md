# 正文重构模板

## 模板定位

这不是把每章写成同一种文章的填空表，而是一套固定的检查顺序。它的目标是让生活语言、专业精度、公开证据和账户动作在同一章里闭环，同时保留不同章节应有的节奏与文学差异。

> 固定的是检查顺序，不是写作腔调。

## 七层检查顺序

### 1. 生活画面

先让读者看见一个普通生活场景，再进入金融概念。

合格标准：

- 画面来自普通人的共同经验；
- 不需要专业背景即可理解冲突；
- 场景天然包含本章的核心变量；
- 不为比喻而比喻。

禁止做法：先抛出术语，再用生活例子装饰定义。

### 2. 经营机制

说明生活画面为什么会发生，以及经营者真正面对的约束。

需要回答：

- 顾客、库存、租金、现金或竞争发生了什么变化？
- 哪个变量是原因，哪个只是结果？
- 规模和时间怎样改变后果？

### 3. 专业映射

把经营语言准确翻译回金融语言。通俗表达是入口，不是对专业概念的替代。

需要写清：

- 对应的专业术语；
- 计算或判断口径；
- 与相邻概念的区别；
- 隐喻不能覆盖的部分。

### 4. 一级命题

每章只保留一条承重命题。它必须能够独立复述，但不能只是漂亮口号。

合格命题同时满足：

- 有认知反转；
- 能被事实验证或反证；
- 能改变研究、仓位或交易动作；
- 一个月后仍容易复述。

### 5. 模型与案例

模型负责规定观察顺序，案例负责让机制接受现实检验。

每个主案例必须说明：

- 它证明本章哪一个机制；
- 使用的是公司披露、监管文件还是其他资料；
- 哪些事实已经核验；
- 它不能证明什么；
- 它与个人股票账户有哪些不可类比之处。

一个案例原则上只有一个主归属章节。其他章节可以简短回指，不重复完整叙述。

### 6. 账户动作

读者看完后必须能改变一个具体动作。可使用卡片、检查表、观察顺序或停止条件，但不提供脱离情境的固定仓位比例和自动买卖指令。

动作至少包含：

- 做什么；
- 不做什么；
- 适用条件；
- 失效条件；
- 主要风险。

### 7. 误用边界

主动写出本章最容易被滥用的方向。边界不是免责声明，而是理论的一部分。

常见误用包括：

- 把隐喻当作精确财务定义；
- 从单一案例推出普遍定律；
- 把长期产业逻辑当作不退出的理由；
- 把模型评分当作自动交易信号；
- 把谨慎原则变成永远不承担风险。

## 五道质量门

| 质量门 | 校对问题 | 不通过时怎样处理 |
| --- | --- | --- |
| 小饭馆老板测试 | 一个经营多年、没学过金融的人能否理解？ | 重写生活画面和经营机制 |
| 专业回译测试 | 专业读者能否准确还原对应概念？ | 补口径、定义和相邻概念边界 |
| 认知反转测试 | 是否真正改变了一个常见直觉？ | 删除空洞金句，重找核心冲突 |
| 行动测试 | 读者明天能改变什么？ | 补动作卡、检查顺序或停止条件 |
| 反证测试 | 什么事实出现时，本章判断不再适用？ | 增加失效条件和案例使用边界 |

## 章节工作卡

每章开工前填写，不写正文也能看出结构是否成立。

| 项目 | 本章答案 |
| --- | --- |
| 唯一任务 | 本章只解决什么问题？ |
| 生活画面 | 读者首先看见什么？ |
| 经营机制 | 画面背后的因果链是什么？ |
| 专业映射 | 对应哪些专业概念和口径？ |
| 一级命题 | 最值得记住的一句话是什么？ |
| 主模型 | 读者按什么顺序观察？ |
| 主案例 | 哪个公开案例最适合检验？ |
| 反例或边界 | 什么情况下不能这样类比？ |
| 账户动作 | 读者具体做什么、不做什么？ |
| 停止条件 | 哪些事实出现时应停止调用本章模型？ |
| 与前章关系 | 承接了什么？ |
| 与后章关系 | 把什么问题留给下一章？ |

## 交稿验收

一章完成，不以字数、案例数或金句数为标准，而以以下结果为标准：

- 不借助术语堆砌也能讲明白；
- 回到专业语言后仍然准确；
- 只有一个清楚的主命题；
- 案例有原始来源和使用边界；
- 读者获得可执行动作；
- 章节之间没有重复承担同一论证；
- 没有公开个人账户、持仓、交易截图和非公开证据。

## 文风约束

- 先写具体，再写抽象；先让人看见，再让人理解。
- 不把段落切成口号墙，完整论证优先于金句密度。
- “不是……而是……”只用于真正的认知反转，不作默认句式。
- 专业术语首次出现时给出人话解释，但不把术语永久删掉。
- 表格只在能显著降低理解成本时使用。
- 结尾回到读者下一次决策，不重复整章摘要。
