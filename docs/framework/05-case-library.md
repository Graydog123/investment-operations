# 验证方法：案例不能替代理论，也不能替代数据

第五册采用‘原始资料 → 事实 → 机制判断 → 反证要求 → 散户动作’五步法。案例可以让机制可见，但一个成功案例不能证明模型普遍成立，一个失败案例也不能自动否定模型。

| 等级 | 含义 | 本册处理 |
| --- | --- | --- |
| A | 原始资料直接支持关键事实，机制映射清晰。 | 可以进入正文主案例；仍保留适用边界。 |
| B | 事实明确，但模型解释含较多推断，或正例证据不完整。 | 可以作为辅助案例；不得写成单因果。 |
| C | 只有局部证据、候选案例或缺少关键市场数据。 | 留在研究池；补证据前不承担一级结论。 |

## 三条使用纪律

> 事实和判断分开<br>‘公司披露了库存减记’是事实；‘库存模型因此成立’是判断。两句话不能合并。

> 反例优先<br>模型如果只能解释成功、不能说明何时停止使用，就还不是决策工具。

> 散户镜像不过度类比<br>银行、基金和企业案例只用于验证机制，不把机构制度、杠杆和时间尺度原样套到个人账户。

序章　投资不是下注，是经营

> 一级命题<br>投资不是预测比赛，而是资本经营。

| 正向／工作案例<br>伯克希尔：现金、经营收益与诚实披露 | 反向／边界案例<br>Zillow Offers：预测误差进入大库存后被放大 |
| --- | --- |
|  |  |

> 正向｜已核验事实<br>伯克希尔在2023年股东信中把GAAP净利润与经营收益分开说明，并强调保持大量现金和短期国债；信中回顾2008年时，公司不依赖商业票据、银行额度或债务市场来满足现金需要。<br>边界：这不能证明高现金在所有时期都最优，也不能把伯克希尔的保险浮存金结构直接复制到个人账户。

> 反向｜已核验事实<br>Zillow在2021年宣布关闭Zillow Offers，并披露库存减记、房价预测不可预测性与运营容量约束；公司同时计划削减约四分之一员工。<br>边界：关闭业务不只由单一模型误差造成，定价、融资、施工和周转能力共同参与。

> 模型判断｜不是事实<br>伯克希尔说明观点之外还必须同时经营现金、负债与资本配置；Zillow说明预测一旦进入大库存和运营约束，账户结果会脱离最初观点。

> 未验证／反证要求<br>尚缺同一主体按完整五层长期记录的公开样本；原名‘四表’与五层结构矛盾。

| 散户账户镜像<br>看好一只股票，却没有同时写仓位、融资、持有理由和现金余量。 | 动作约束<br>买入前补齐观点、资产、负债、库存理由和现金五栏；缺一栏就降仓。 |
| --- | --- |
| 原模型适用边界<br>它不否定公司研究和价格判断；四表只是把判断放进真实账户，不能替代具体行业研究。 | 停止使用<br>若四张经营表无法从真实账户数据中重建，模型暂停使用，先修复数据口径。 |

| 阶段结论　保留模型；下一版改名为‘观点卡—四张经营表’。 | 来源　S01、S07 |

> 一级命题<br>成交量说明筹码换得多快，不说明最后站在哪一边。

| 正向／工作案例<br>GameStop：热闹是真的，单一解释却不够 | 反向／边界案例<br>GameStop：热闹是真的，单一解释却不够 |
| --- | --- |
|  |  |

> 正向｜已核验事实<br>SEC工作人员报告记录了2021年GameStop价格、成交、空头兴趣、社交媒体关注和零售交易限制等多重因素。<br>边界：报告不支持把涨跌完全归因于单一群体或单一机制；它更适合检验量价解释的边界。

> 同一案例的双重作用<br>该案例既支持模型，也限制模型：热闹与换手真实存在，但单一因果和方向预测无法从成交量直接推出。

> 模型判断｜不是事实<br>同一案例同时出现价格、成交、空头、社交传播与交易限制，恰好说明成交量只能描述换手，不能单独给出方向或因果。

> 未验证／反证要求<br>缺少系统化样本检验四象限在不同市场阶段的预测能力。

| 散户账户镜像<br>看到放量上涨就认定机构抢筹，看到放量下跌就认定主力出货。 | 动作约束<br>量价只做分类器；必须再看位置、收盘、次日承接和板块共振。 |
| --- | --- |
| 原模型适用边界<br>重大公告、涨跌停和极端流动性会扭曲成交含义，需要单独解释。 | 停止使用<br>若行情数据、复权或停复牌状态异常，暂停矩阵判断。 |

| 阶段结论　保留为诊断工具，不得包装成预测公式。 | 来源　S14 |

> 正向｜已核验事实<br>Target在2022年公布加大折价、移除过剩库存、取消订单和调整品类等措施，以重新匹配需求并改善后续经营。<br>边界：主动清库存会损伤短期利润，不能仅凭采取行动就断言调整一定成功。

> 反向｜已核验事实<br>Zillow在2021年宣布关闭Zillow Offers，并披露库存减记、房价预测不可预测性与运营容量约束；公司同时计划削减约四分之一员工。<br>边界：关闭业务不只由单一模型误差造成，定价、融资、施工和周转能力共同参与。

> 未验证／反证要求<br>主动处理库存不保证后续成功；股票持仓也没有实物腐败的完全等价关系。

| 散户账户镜像<br>短线催化失效后，用‘公司长期不错’给原仓改名。 | 动作约束<br>每次理由改变都视为新交易，重新决定价格、规模和退出。 |
| --- | --- |

> 未验证／反证要求<br>制造业停线与投资卖出不同，投资中的异常阈值必须结合策略而定。

隆基绿能：2022—2024年的扩产、价格下跌、减值与利润反转

> 研究问题<br>这组A股材料是否真正补上原模型缺失的证据，而不是只提供一个熟悉的公司故事？

> 已核验事实• 公司披露材料显示，2022年需求旺盛时营业收入与归母净利润均明显增长，经营活动现金流同步改善。<br>• 公司随后披露，2023年下半年行业集中扩产、硅料由紧缺转向过剩，硅片与组件价格快速下行。<br>• 隆基硅片产能从2021年的96.61GW增至2023年的146.59GW；公司2023、2024年分别计提大额存货等减值。<br>• 公司材料列示母公司口径净利润由2022年的148.12亿元、2023年的107.51亿元，转为2024年的−86.18亿元。

> 模型判断｜不是事实<br>这组连续披露把‘需求旺盛—扩产—供给宽松—价格下跌—减值—利润反转’连成了一条可复核链。它足以证明，长期需求增长不能阻止中期供给周期伤害利润。

> 反证与边界<br>样本仍未完整覆盖弱产能退出、行业集中度回升和利润重新修复，因此只验证到‘过剩与利润压缩’，没有验证完‘出清与复苏’。

| 散户怎么用<br>研究光伏、锂电、存储等周期成长行业时，不能只看终端需求。至少同步记录在建产能、投产节奏、库存、产品价格、减值和现金流。 | 下一步怎么证伪<br>补2025—2027年产能退出、开工率、龙头资本开支和价格修复数据，完成一次闭环。 |

# v0.3：从案例库走向可累计的实验室

> 下一步不是继续找故事<br>v0.3应让案例能够被复算、被反驳、被长期更新。优先补公司原始资料与统一数据口径，不再扩张模型数量。

> 建立统一行情口径<br>统一前复权、分红、估值、相对强弱和板块分类，避免M19等模型因数据口径变化得出不同结论。

> 建立失败池<br>每个模型同时保存支持、反例和停止使用记录；模型被反例击中时必须降级。

> 补跨周期样本<br>M07补出清复苏，M08补三轮主线，M19补多公司，M27补半导体和创新药横截面。

> 与正文建立双向链接<br>正文引用案例编号；案例卡反向标注正文命题。事实变化时只更新案例库，不让正文悄悄改写历史。

四个C级模型已经被推进到B级，但它们仍然可以在新证据面前降回C级。能被降级，才说明这不是一套只会证明自己的理论。
