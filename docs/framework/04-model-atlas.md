# 《模型图谱》

*30个把命题变成判断的核心模型*

“命题先决定我们相信什么，模型再说明它怎样运转、何时能用、何时必须停止。”

生活画面 → 经营机制 → 变量关系 → 可观察证据 → 决策动作

v0.1 · 模型初稿 · 继续迭代 · 不构成任何个股买卖建议

# 一页总览：模型不是答案，是一组有顺序的问题

第四册承接第三册的30条一级命题。每个模型必须同时做到五件事：能画、能讲、能观察、能行动、能停止。只提供漂亮解释却不能指导一次真实判断的图，不进入模型库。

| 先看画面 | 用普通人见过的餐馆、房租、库存和客流建立直觉。 |
| --- | --- |
| 再看机制 | 明确什么变量推动什么变量，方向和反馈如何发生。 |
| 寻找证据 | 把抽象概念还原成价格、成交、现金、债务和经营数据。 |
| 决定动作 | 模型最终必须改变研究、仓位、融资、等待、退出或复盘。 |
| 设置停线 | 数据失真、边界失效或证据不足时，模型必须允许自己停止。 |

## 六组模型

| 第一组　账户与生存<br>先解决能不能继续开店，再讨论下一轮赚多少。 | 第二组　主线与组合<br>从街区、客流和供给看懂市场选择。 |
| --- | --- |
| 第三组　证据与仓位<br>让证据升级、仓位扩张和规则执行彼此匹配。 | 第四组　复盘与风险<br>把损失翻译成制度，把波动与失去选择区分开。 |
| 第五组　估值与资本<br>从价格要求的未来，追到真正属于股东的现金。 | 第六组　退出与系统<br>让资本能够退出、重新进入，并通过真实记录持续学习。 |

使用顺序：先选问题所在的层级，再打开对应模型；不要用仓位模型替代公司研究，也不要用公司研究替代账户生存判断。

# 模型语法：每张图必须说清六件事

| 变量<br>什么东西正在变化。 | 方向<br>A增加时，B通常怎样变化。 |
| --- | --- |
| 反馈<br>变化会自我强化，还是被某种力量制衡。 | 证据<br>现实中看什么数据，才能知道模型正在运转。 |
| 动作<br>模型成立后，研究、仓位或退出具体改变什么。 | 停线<br>什么条件出现时，不再允许模型继续解释现实。 |

## 三个使用禁区

| 不要单因果 | 模型是压缩现实，不是宣布世界只有一个原因。 |
| --- | --- |
| 不要静态化 | 变量、权重和反馈会随市场阶段变化。 |
| 不要替代证据 | 生活画面帮助理解，不能代替公告、财报、价格和账户数据。 |

一张图只有在读者能够指出它可能错在哪里时，才真正开始有用。

# M01 投资经营四表模型

> 投资不是预测比赛，而是资本经营。

生活画面　一家餐馆不能只看招牌菜好不好，还要同时看账上有多少钱、欠了多少房租、冰箱压了多少货，以及今天还能不能继续开门。

模型任务　把“我看对了吗”改写为“即使看错，账户会怎样”，让观点、资产、负债、库存与现金进入同一张经营图。

### 关键变量

| 观点质量 | 事实与假设的可靠程度 |
| --- | --- |
| 资产规模 | 上涨与下跌对净值的放大 |
| 负债期限 | 等待时间与强制行动压力 |
| 库存状态 | 持有理由是否仍然新鲜 |
| 现金余量 | 应对冲击和新机会的能力 |

M01 操作卡　|　序章　投资不是下注，是经营

# 模型怎样运转

| STEP 1<br>观点只提出可能的方向，不能单独决定账户结果。 | STEP 2<br>资产和仓位把观点翻译成盈亏尺寸。 |
| --- | --- |
| STEP 3<br>负债和资金期限决定你能等多久。 | STEP 4<br>库存与现金决定错误发生后能否调整。 |

## 从观察到行动

| 证据仪表盘 | 决策动作 |
| --- | --- |
| 观察 1　同一方向的总暴露而非股票数量 | 动作 1　买入前同时写下观点、规模、资金期限与退出。 |
| 观察 2　融资余额、到期日和保证金余量 | 动作 2　每周把账户按资产、负债、库存、现金重做一次经营盘点。 |
| 观察 3　持仓理由与原始记录是否一致 | 动作 3　发现任何一张表恶化时，先降低规模，再讨论观点。 |
| 观察 4　卖出后是否仍有可用资本 |  |

## 使用边界

| 适用条件 | 适用于所有主动投资账户，尤其适合集中持仓、融资和高换手账户。 |
| --- | --- |
|  |  |

| 误用边界 | 它不否定公司研究和价格判断；四表只是把判断放进真实账户，不能替代具体行业研究。 |
| --- | --- |
|  |  |

| 停止条件 | 若四张经营表无法从真实账户数据中重建，模型暂停使用，先修复数据口径。 |
| --- | --- |
|  |  |

# M02 经营权模型

> 账户的首要产出不是收益，而是下一次配置资本的能力。

生活画面　一家店某个月赚得很多，却把现金、信用和老板的身体全部耗尽；它看起来盈利，实际上可能已经失去继续经营的资格。

模型任务　用“下一轮还能不能自主决策”评价收益质量，把流动性、信用和心理资本纳入账户产出。

### 关键变量

| 净资本 | 可承受下一次试错的本金 |
| --- | --- |
| 流动性 | 能否不被迫卖出 |
| 信用余量 | 债权人是否夺走决策权 |
| 心理资本 | 是否还能睡觉、研究和执行 |
| 机会承载力 | 新机会出现时能否投入 |

M02 操作卡　|　序章　投资不是下注，是经营

# 模型怎样运转

| STEP 1<br>收益增加本金，但风险结构可能同时消耗自由。 | STEP 2<br>自由减少会让下一次决策被债务或情绪接管。 |
| --- | --- |
| STEP 3<br>账户只有保留资本与判断力，收益才能进入下一轮。 | STEP 4<br>复利的前提不是每次赚钱，而是经营权不断续期。 |

## 从观察到行动

| 证据仪表盘 | 决策动作 |
| --- | --- |
| 观察 1　盈利后融资是否下降 | 动作 1　把经营权列为账户一级指标，与收益率并列。 |
| 观察 2　回撤后可用现金和保证金是否仍充足 | 动作 2　大赚后优先恢复现金、信用和心理余量。 |
| 观察 3　仓位是否影响睡眠与执行 | 动作 3　任何可能永久削弱经营权的单笔交易都降级处理。 |
| 观察 4　新机会出现时是否只能卖旧仓换新仓 |  |

## 使用边界

| 适用条件 | 适用于追求长期复利的个人账户，也适用于经历大赚、大亏或高杠杆之后的重建阶段。 |
| --- | --- |
|  |  |

| 误用边界 | 保留经营权不是永远轻仓；当证据、赔率和退出条件同时优秀时，资本可以主动扩张。 |
| --- | --- |
|  |  |

| 停止条件 | 若生活资金、投资资金和债务无法分离，停止讨论收益目标，先恢复基本经营边界。 |
| --- | --- |
|  |  |

# M03 价格—账户双层现实

> 市场可以错，但账户必须尊重市场。

生活画面　店主可以认为顾客误解了招牌菜，但今天没人来，收银台就是没有钱；房租不会因为店主最终正确而暂停。

模型任务　把价值判断与账户约束分层：允许保留观点，同时要求行动尊重价格、流动性和资金期限。

### 关键变量

| 内在价值假设 | 长期经营结果 |
| --- | --- |
| 市场价格 | 当前支付意愿 |
| 资金期限 | 观点等待窗口 |
| 流动性 | 退出的真实成本 |
| 仓位 | 偏离期间的净值压力 |

M03 操作卡　|　第一章　顾客是上帝

# 模型怎样运转

| STEP 1<br>价值与价格可以偏离，偏离时间却不可预知。 | STEP 2<br>账户的债务和期限把时间变成成本。 |
| --- | --- |
| STEP 3<br>仓位越大，价格现实越快影响判断和行动。 | STEP 4<br>成熟做法是保留观点、调整承诺，而不是争论谁真谁假。 |

## 从观察到行动

| 证据仪表盘 | 决策动作 |
| --- | --- |
| 观察 1　价格对利好与利空的反应 | 动作 1　把价值结论和交易结论分开记录。 |
| 观察 2　相对板块强弱和成交承接 | 动作 2　价格弱势时同时审查公司、市场和账户，不只重读基本面。 |
| 观察 3　融资与生活资金期限 | 动作 3　等待权不足时用减仓购买时间。 |
| 观察 4　结构止损到当前价的距离 |  |

## 使用边界

| 适用条件 | 适用于价值判断与市场走势冲突的阶段，尤其适合高波动、高仓位和有融资的账户。 |
| --- | --- |
|  |  |

| 误用边界 | 尊重市场不等于股价一跌就卖，也不等于短期价格永远正确。 |
| --- | --- |
|  |  |

| 停止条件 | 若价格异常来自停牌、数据错误或流动性中断，暂不使用相对强弱结论。 |
| --- | --- |
|  |  |

# M04 客流—翻台矩阵

> 成交量说明筹码换得多快，不说明最后站在哪一边。

生活画面　餐馆突然坐满，可能是新顾客涌入，也可能是老顾客吃完最后一顿离场；热闹本身不能说明明天还会不会排队。

模型任务　把价格方向与成交活跃度交叉判断，区分有效突破、健康整理、派发和失去兴趣。

### 关键变量

| 价格方向 | 顾客支付意愿变化 |
| --- | --- |
| 成交额 | 筹码周转速度 |
| 收盘位置 | 当天谁取得主动 |
| 趋势阶段 | 启动、中继、加速或退潮 |
| 后续承接 | 热闹是否具有连续性 |

M04 操作卡　|　第二章　股价是客流，成交量是翻台

# 模型怎样运转

| STEP 1<br>成交同时存在买方和卖方，不自带方向。 | STEP 2<br>价格位置把换手放进趋势上下文。 |
| --- | --- |
| STEP 3<br>收盘与次日反馈验证供给是否被吸收。 | STEP 4<br>同样的放量在不同阶段可能含义相反。 |

## 从观察到行动

| 证据仪表盘 | 决策动作 |
| --- | --- |
| 观察 1　收盘在日内振幅中的位置 | 动作 1　放量时先标记阶段，再解释成交。 |
| 观察 2　成交额相对5日、20日均值 | 动作 2　将突破、滞涨、下跌三类放量分别记录。 |
| 观察 3　突破后1—3日是否守住平台 | 动作 3　没有后续承接，不把一天热闹升级为主线。 |
| 观察 4　板块核心是否共振 |  |

## 使用边界

| 适用条件 | 适用于趋势、主线轮动和突破交易的盘后复盘。 |
| --- | --- |
|  |  |

| 误用边界 | 重大公告、涨跌停和极端流动性会扭曲成交含义，需要单独解释。 |
| --- | --- |
|  |  |

| 停止条件 | 若行情数据、复权或停复牌状态异常，暂停矩阵判断。 |

# M05 库存腐败链

> 真正腐败的不是股票，而是继续持有它的理由。

生活画面　牛肉不会因为老板高价进货就停止变质；越舍不得承认买多了，冰箱里的损失越可能从折价处理变成全部报废。

模型任务　追踪持仓理由从新鲜证据滑向沉没成本、叙事漂移和被动等待的过程。

### 关键变量

| 理由保质期 | 催化和趋势的有效时间 |
| --- | --- |
| 新增证据 | 支持或反对原假设的事实 |
| 机会成本 | 资本错过的替代选择 |
| 流动性 | 库存折价处理难度 |
| 叙事漂移 | 当前理由与买入理由的距离 |

# 初稿之后

这30个模型已经把第三册的一级命题第一次转换成变量、方向、反馈、证据、动作和停止条件。但它们仍是工作模型，不是不可修改的教条。

下一轮需要做三类验证：用历史案例检查解释力，用真实账户检查行动价值，用反例检查边界。凡是无法改善一次真实判断的模型，都应合并、降级或退出模型库。

第五阶段回到正文时，模型不会被当作插图。每章应先让读者在生活中看见问题，再用模型解释机制，最后回到金融语言与投资动作。

模型不是为了让世界看起来简单，而是为了让复杂世界中的行动更诚实。
